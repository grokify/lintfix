## [v0.1.0] - 2026-03-07

### Highlights

- Structured lint remediation database mapping golangci-lint rules to fix strategies
- Nolint comment generators with pre-written reason strings for common scenarios

### Added

- Remediation database with support for gosec, staticcheck, and errcheck rules ([`45810d6`](https://github.com/grokify/lintfix/commit/45810d6))
- `LoadRemediations()` and `MustLoadRemediations()` functions to load embedded database ([`45810d6`](https://github.com/grokify/lintfix/commit/45810d6))
- `RemediationDB.Get()`, `GetGosec()`, `GetStaticcheck()` methods for querying rules ([`45810d6`](https://github.com/grokify/lintfix/commit/45810d6))
- `RuleFix.HasHelper()` method to check if a code-based helper function exists ([`45810d6`](https://github.com/grokify/lintfix/commit/45810d6))
- `gosec.Nolint()` generic function for generating nolint:gosec comments ([`45810d6`](https://github.com/grokify/lintfix/commit/45810d6))
- `gosec.NolintG117()`, `NolintG118()`, `NolintG704()`, `NolintG101()` rule-specific generators ([`45810d6`](https://github.com/grokify/lintfix/commit/45810d6))
- `gosec.CommonReasons` struct with pre-written reason strings for OAuth, shutdown handlers, SSRF, etc. ([`45810d6`](https://github.com/grokify/lintfix/commit/45810d6))
- JSON schema for remediation database with type, summary, pattern, package, function, example fields ([`45810d6`](https://github.com/grokify/lintfix/commit/45810d6))

### Documentation

- Package documentation with usage examples for database queries and nolint generation ([`45810d6`](https://github.com/grokify/lintfix/commit/45810d6))
- README with installation, usage examples, and supported linters reference ([`45810d6`](https://github.com/grokify/lintfix/commit/45810d6))
